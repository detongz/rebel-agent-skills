## Submission Process

### How to Submit
1. Visit the official hackathon website: **https://moltiverse.dev**
2. Register your account/team
3. Submit your project directly through the platform
4. Include all required information (see below)

### Submission Requirements

#### For **Agent + Token Track**:
- Deploy a token on nad.fun
- Include the token address in your submission
- Agent must interact with the token
- Token becomes part of the ecosystem

#### For **Agent Track** (No Token Required):
- Working agent that does something interesting
- Monad integration optional but recommended
- Clear demo or documentation
- No blockchain experience required

## Required Information for Submission

Based on the hackathon documentation, submissions should include:

1. **Project Description**
   - What your agent does
   - How it works
   - What makes it unique

2. **Demo/Documentation**
   - Working demo or clear documentation
   - Instructions for testing

3. **Track-Specific Requirements**:
   - **Agent+Token Track**: Token address on nad.fun
   - **Agent Track**: Monad integration details (if applicable)

4. **Code Attribution**

## FAQ

**Q: Do I need crypto experience?**
A: No! Choose the Agent Track if you're new to blockchain. You'll only need basic Monad integration for logging or coordination.

**Q: Can I submit to both tracks?**
A: Yes, but projects must be substantially different.

**Q: Do I need a team?**
A: No! Solo developers and teams are both welcome.

**Q: Can I use existing code?**
A: Yes, but clearly document what's original vs. what's reused. The innovation must be substantial.

**Q: When will I know if I won?**
A: Winners are contacted prior to announcement. Rolling judging means early submissions can get early results.

**Q: How many winners total?**
A: Up to 16 winners ($10K each) + 1 liquidity boost winner ($40K).
